# ✅ 域名配置检查清单

## 📋 配置前准备

- [ ] 域名已在阿里云注册
- [ ] 已登录阿里云控制台
- [ ] 已登录 Vercel Dashboard
- [ ] 项目已部署到 Vercel（有 .vercel.app 地址）

## 🔧 Vercel 配置

- [ ] 在 Vercel Dashboard 进入项目设置
- [ ] 进入 Settings → Domains
- [ ] 点击 "Add Domain"
- [ ] 输入域名（例如：`stock.example.com`）
- [ ] 点击 "Add"
- [ ] 记录 Vercel 提供的 DNS 配置信息

## 🌐 阿里云 DNS 配置

### 方式一：直接使用阿里云 DNS（简单）

- [ ] 登录阿里云控制台
- [ ] 进入 域名 → 域名解析
- [ ] 选择你的域名
- [ ] 添加 CNAME 记录：
  - [ ] 记录类型：CNAME
  - [ ] 主机记录：@ 或 www（根据需求）
  - [ ] 记录值：`cname.vercel-dns.com`
  - [ ] TTL：600（或默认）
- [ ] 点击确认保存

### 方式二：使用 Cloudflare（推荐，CDN加速）

- [ ] 登录 Cloudflare Dashboard
- [ ] 添加站点（Add a Site）
- [ ] 输入域名
- [ ] 选择免费计划
- [ ] 获取 Cloudflare DNS 服务器地址
- [ ] 在阿里云修改 DNS 服务器为 Cloudflare 提供的地址
- [ ] 在 Cloudflare 配置 CNAME 记录：
  - [ ] Type: CNAME
  - [ ] Name: @ 或 www
  - [ ] Target: `cname.vercel-dns.com`
  - [ ] Proxy: 🟠 Proxied（橙色云朵）
- [ ] 保存配置

## 🔒 SSL 证书

- [ ] 等待 DNS 解析生效（5-30分钟）
- [ ] 在 Vercel Dashboard 查看域名状态
- [ ] 等待 SSL 证书自动配置（几分钟到几小时）
- [ ] 确认状态显示 "Valid Configuration"

## ✅ 验证测试

- [ ] 使用 `dig` 或 `nslookup` 检查 DNS 解析
- [ ] 在浏览器访问域名
- [ ] 确认可以正常访问
- [ ] 确认显示 HTTPS（🔒 锁图标）
- [ ] 测试主要功能是否正常

## 🐛 问题排查

如果配置后无法访问：

- [ ] 检查 DNS 解析是否正确（使用 dig/nslookup）
- [ ] 检查 Vercel 域名状态
- [ ] 检查 Vercel 部署是否成功
- [ ] 清除浏览器缓存
- [ ] 清除本地 DNS 缓存
- [ ] 等待更长时间（DNS 可能需要 24-48 小时完全生效）
