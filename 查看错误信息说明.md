# 如何查看服务器错误信息

## 方法1：查看运行服务的终端窗口（最简单）

1. 找到运行 `python3 surge_analysis_web.py` 的终端窗口
2. 错误信息会直接显示在那个窗口中
3. 当你在网页上点击"添加股票"时，终端会显示：
   - 请求信息
   - 错误堆栈（如果有错误）
   - 完整的错误跟踪

## 方法2：重新在前台运行服务（推荐用于调试）

1. 停止当前服务：
   ```bash
   pkill -f surge_analysis_web.py
   ```

2. 在前台运行（这样可以看到所有输出）：
   ```bash
   python3 surge_analysis_web.py
   ```

3. 现在所有输出都会显示在这个终端窗口中
4. 当你在网页上操作时，错误信息会立即显示

## 方法3：使用日志文件

1. 启动服务时重定向输出到文件：
   ```bash
   python3 surge_analysis_web.py > web.log 2>&1 &
   ```

2. 查看日志：
   ```bash
   tail -f web.log
   ```

3. 或者查看最近的错误：
   ```bash
   tail -50 web.log | grep -A 10 "error\|Error\|Exception"
   ```

## 方法4：使用我提供的命令

运行以下命令查看最近的错误：
```bash
tail -100 web.log 2>/dev/null | grep -A 20 "error\|Error\|Exception\|Traceback" | tail -50
```

如果没有日志文件，服务可能是在前台运行的，直接看终端窗口即可。
