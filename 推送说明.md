## 如果推送需要认证

### 使用Personal Access Token（推荐）

1. 在GitHub上创建Personal Access Token：
   - 访问：https://github.com/settings/tokens
   - 点击 "Generate new token (classic)"
   - 选择权限：`repo`（完整仓库权限）
   - 复制生成的token

2. 推送时使用token：
   ```bash
   git push https://<YOUR_TOKEN>@github.com/jet20002025-hash/daniugu.git local-dev-stock-search
   ```

3. 或者配置credential helper：
   ```bash
   git config --global credential.helper osxkeychain
   git push origin local-dev-stock-search
   # 输入用户名和token（不是密码）
   ```

### 使用SSH密钥

1. 检查是否已有SSH密钥：
   ```bash
   ls -la ~/.ssh
   ```

2. 如果没有，生成SSH密钥：
   ```bash
   ssh-keygen -t ed25519 -C "your_email@example.com"
   ```

3. 添加SSH密钥到GitHub：
   - 复制公钥：`cat ~/.ssh/id_ed25519.pub`
   - 访问：https://github.com/settings/keys
   - 点击 "New SSH key"，粘贴公钥

4. 修改远程URL为SSH：
   ```bash
   git remote set-url origin git@github.com:jet20002025-hash/daniugu.git
   git push origin local-dev-stock-search
   ```

## 常见问题

### 1. 推送被拒绝（403 Forbidden）
- 原因：认证失败或权限不足
- 解决：检查token/SSH密钥配置，确保有推送权限

### 2. 推送被拒绝（non-fast-forward）
- 原因：远程分支有本地没有的提交
- 解决：先拉取远程更改 `git pull origin local-dev-stock-search --rebase`

### 3. 网络超时
- 原因：网络连接问题
- 解决：检查网络连接，或使用代理
